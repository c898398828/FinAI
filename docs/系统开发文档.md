# 系统开发文档

## 1. 项目概述

F-AI 财务助手是一个面向企业管理者与财务人员的 Web 系统，提供：

- 财务数据导入与记录管理
- 财务报表生成与导出
- 异常预警与预算管理
- AI 对话与财务分析
- 企业成员与权限管理

---

## 2. 架构设计

### 2.1 整体架构

- 前端（Vue3）：负责页面展示、交互、图表渲染、权限路由拦截
- 后端（FastAPI）：提供鉴权、业务逻辑、数据持久化、AI 调用
- 数据层：
  - MySQL：业务数据（用户、企业、财务记录、预算、报表、预警）
  - SQLite：系统配置（LLM 配置持久化）

### 2.2 目录说明

```text
backend/app/api        # 接口层
backend/app/services   # 业务服务层
backend/app/models     # ORM 模型
backend/app/schemas    # Pydantic 模型
backend/app/utils      # 工具类（安全、Excel 解析等）
frontend/src/views     # 页面视图
frontend/src/api       # 前端 API 封装
frontend/src/stores    # 状态管理
frontend/src/router    # 前端路由与权限守卫
```

## 3. 开发环境

### 3.1 依赖要求

- Python 3.10+
- Node.js 18+（建议 18/20 LTS）
- MySQL 8.x

### 3.2 启动步骤

#### 后端

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### 前端

```bash
cd frontend
npm install
npm run dev
```

前端默认代理：`/api -> http://localhost:8000`

## 4. 配置说明

### 4.1 后端环境变量

文件：`backend/.env`

| 变量 | 说明 | 默认值 |
|---|---|---|
| `DATABASE_URL` | MySQL 连接串 | `mysql+pymysql://root:password@localhost:3306/fai_db` |
| `SECRET_KEY` | JWT 密钥 | `dev-secret-key-change-in-production` |
| `ALGORITHM` | JWT 算法 | `HS256` |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | Token 过期时间（分钟） | `1440` |
| `LLM_API_KEY` | 大模型 API Key | 空 |
| `LLM_BASE_URL` | 大模型 Base URL | `https://api.deepseek.com/v1` |
| `LLM_MODEL` | 大模型名称 | `deepseek-chat` |

### 4.2 系统配置持久化（SQLite）

新增存储模块：`backend/app/services/system_config_store.py`

- 数据文件：`backend/data/system_config.db`
- 数据表：`system_config`
- 当前持久化键：
  - `llm.api_key`
  - `llm.base_url`
  - `llm.model`

说明：
- LLM 配置读取优先级：运行时缓存 > SQLite > `.env`
- `/api/ai/config` 保存后重启服务仍保留

## 5. 权限模型

### 5.1 角色定义

- 超级管理员：`company_super_admin = true`
- 企业管理员：`role = admin`
- 其他成员：`role = accountant | viewer`

### 5.2 权限规则

- 仅超级管理员可：
  - 设置企业管理员
  - 转移超级管理员
  - 解散企业
- 超级管理员/企业管理员可：
  - 邀请成员加入企业
  - 移除其他成员（不可移除超级管理员）
  - 分配成员权限（企业管理员不可把成员设为 `admin`）
- 普通成员不可：
  - 管理企业
  - 分配权限

### 5.3 前端限制

文件：`frontend/src/router/index.ts`、`frontend/src/components/layout/AppSider.vue`

- 非企业管理角色不可访问 `/company`、`/permissions`
- 菜单层隐藏组织管理入口

## 6. 核心接口清单

## 6.1 Auth（`/api/auth`）

- `POST /register` 注册
- `POST /login` 登录
- `GET /me` 当前用户信息
- `POST /company` 创建企业
- `POST /company/join` 加入企业

## 6.2 Users（`/api/users`）

- `GET /` 用户列表（管理员）
- `PUT /{user_id}` 用户更新
- `GET /company/members` 企业成员列表
- `POST /company/members/invite` 邀请成员
- `PUT /company/members/{user_id}/role` 设置成员角色
- `DELETE /company/members/{user_id}` 移除/退出企业
- `POST /company/super-admin/transfer` 转移超级管理员
- `DELETE /company` 解散企业

## 6.3 Financial（`/api/financial`）

- `POST /upload` 批量导入
- `GET /records` 记录列表
- `POST /records` 手动新增记录
- `GET /summary` 汇总
- `GET /trend` 收入趋势
- `GET /cashflow` 现金流
- `GET /template` 导入模板下载

## 6.4 Reports（`/api/reports`）

- `POST /generate` 生成报表
- `GET /` 报表列表
- `GET /{report_id}` 报表详情
- `GET /{report_id}/export` 报表导出

## 6.5 AI（`/api/ai`）

- `POST /anomaly-detect` 异常检测
- `POST /forecast` AI 预测
- `POST /budget-suggest` AI 预算建议
- `GET /dashboard-insights` 仪表盘 AI 洞察
- `POST /chat` AI 对话
- `POST /chat/stream` AI 流式对话
- `POST /analyze-report` AI 报表分析
- `GET /config` 读取 AI 配置
- `PUT /config` 保存 AI 配置
- `POST /config/test` 测试 AI 连接

## 7. 数据库设计（简要）

业务核心表（MySQL）：

- `users`：用户、角色、企业归属、超级管理员标记
- `companies`：企业信息

- `system_config(config_key, config_value, updated_at)`

## 9. 联调与验证建议

### 9.1 基础验证

- 登录后访问仪表盘、导入、报表、AI 页面
- 上传模板数据后验证趋势、现金流图表展示
- 生成报表并执行导出

### 9.2 权限验证（重点）

- 超级管理员：可设置企业管理员、可访问组织与权限管理
- 企业管理员：可邀请/移除成员、可分配非超级管理员权限
- 普通成员：不可见组织菜单，直接访问相关路由会被重定向

### 9.3 AI 配置持久化验证

1. 在系统设置中保存 Base URL / Key / Model
2. 重启后端服务
3. 再次读取配置确认已持久化
4. 检查 SQLite：`backend/data/system_config.db`

## 10. 常见问题

### 10.1 前端 `npm run build` 报 `vue-tsc` 错误

现象：`Search string not found: "/supportedTSExtensions = .*(?=;)/"`

建议：
- 锁定兼容的 Node / TypeScript / vue-tsc 版本
- 先用 `npx vite build` 验证打包链路是否正常

### 10.2 AI 配置测试返回失败但前端提示成功

已修复为基于 `success` 字段显示结果。  
后端返回示例：

```json
{
  "success": false,
  "message": "连接失败: ...",
  "latency_ms": 1855,
  "model": "xxx"
}
```

前端应显示失败提示。
